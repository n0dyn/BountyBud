XSS Testing Guide

Testing Methodology
  1. Identify input points and injection contexts
  2. Start with basic payloads to test for filtering
  3. Use context-specific payloads based on where your input appears
  4. Try bypass techniques if basic payloads are blocked
  5. Use encoding if special characters are filtered
  6. Test with blind XSS payloads for admin panels
Common Filters & Bypasses
  • Script tag blocking: Use event handlers or other tags
  • Case sensitivity: Try mixed case variations
  • Keyword filtering: Use encoding or alternative functions
  • Angle bracket filtering: Use existing tags or attributes
  • Quote filtering: Use different quote types or escape sequences
XSS Types
  • Reflected XSS: Payload executed immediately from request
  • Stored XSS: Payload stored and executed for other users
  • DOM XSS: Payload executed via DOM manipulation
  • Blind XSS: Payload executed in contexts you can't see
