## I. OAuth 2.0 Advanced Exploitation

### 1. Cross-App Account Takeover (COAT)
In ecosystems with multiple OAuth clients, attempt to use an authorization code or token granted for a "Low Privilege App" to authenticate to a "High Privilege App".
- Swap `client_id` during the token exchange or authorization flow.

### 2. OIDC Token Forgery / Swapping
If an application uses an OpenID Connect `id_token` to establish a session:
- **Missing `aud` Verification:** Pass an `id_token` generated by a completely different application you control.
- **Missing `nonce` Verification:** Replay an old `id_token`.
- **Algorithm Confusion:** Change `RS256` to `HS256` and sign with the IdP's public key.

### 3. Redirect URI Bypasses
- **Path Traversal:** `redirect_uri=https://trusted.com/callback/../attacker/`
- **Parameter Injection:** `redirect_uri=https://trusted.com/callback?redirect=https://attacker.com`
- **Regex Flaws:** `trusted.com.attacker.com` or `attacker.com?trusted.com`

### 4. Dynamic Client Registration Abuse
If `POST /register` is open, register a malicious client.
- Set `logo_uri` or `jwks_uri` to an internal AWS/GCP metadata IP to trigger SSRF when the Authorization Server fetches them.

## II. SAML 2.0 Advanced Exploitation

### 1. XML Signature Wrapping (XSW)
Provide a valid signature but "wrap" it around a forged assertion.
- **XSW1:** Add a cloned unsigned copy of the Response after the existing signature.
- **XSW3:** Add a cloned unsigned copy of the Assertion before the existing Assertion.
- The validator verifies the signature on the original, but the business logic parses the forged assertion.

### 2. XML Comment Truncation
Inject comments into the `NameID` field to bypass signature validation while tricking the parser.
- Payload: `admin<!-- -->@victim.com`
- If the signature validates the whole string, but the application parses up to the comment, you log in as `admin`.

### 3. Signature Stripping
Remove the entire `<Signature>` XML block. Some Service Providers (SPs) default to a "success" state if no signature is present to fail.
